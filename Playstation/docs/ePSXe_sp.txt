***********************************************************************
0. Introducción
***********************************************************************

ePSXe es un emulador de la consola de videojuegos Sony Playstation
sobre un PC, bajo el sistema operativo Windows Vista+ o Linux. Que 
utiliza los plugins desarrollados para el psemupro.

***********************************************************************
1. Nuevo en esta versión.
***********************************************************************

ePSXe v2.0.5, released on 06.23.2016

- Actualizado el recompilador para X86 usandose ahora el mismo que en el
  resto de versiones (Android, Linux x64, MacOSX ...). Este facilitará
  arreglar problemas en todas las versiones.

- Portada el local netplay desde Android (Experimental). Esta version es
  compatible usando local netplay con el resto de verisiones de ePSXe
  de la 2.0.5. (MacOSX, Linux x86, Linux x64 y Android - version 2.0.6).

- Portado desde Android el modo threading del GPUCORE. Las CPUs de los PC
  son mas rapidas que las de Android, de modo que la mejora de FPS es menor.
  (Esta opción funcionará mejor cuanto más lenta sea la CPU).

- Añadidos nuevos modos de threading al Plugin Pete OpenGL2 Core. Se
  incluye un modo 2-thread basico, y 2 modos extras que añaden varios
  efectos gráficos. (Esto puede mejorar algunos efectos en el juego, por
  ejemplo la escena de la foto que se queda al inicio del Castlevania).

- Mejoras en el plugin INPUTCORE, corregidos la vibracion en el modo
  Xinput ya que estaban invertidos los motores. Además cuando se juegue
  en modo digital, tambien se puede usar el stick izquierdo como si
  fuera el DPAD.

- Arreglado el soporte para roms PBP mayores de 2GB. Now is posible usar
  roms creadas con PSX2PSP mayores de 2G como el pack del Final Fantasy VIII.

- Arreglado el soporte de cheat codes en modo interpreter.

- Incluida una nueva opcion en la ventana de descarga de los cheat codes 
  para poder descargarlos desde el servidor (Gracias a psxdatacenter.com)

- Arreglos en el core que solucionan problemas en varios juegos.
     - WING COMMANDER IV
     - MONKEY HERO

3.1. Qué es lo que se necesita para ejecutar ePSXe?

* ePSXe.exe
* una bios de psx(scph1001.bin, scph7502.bin has sido testeadas y funcionan)
* un plugin de vídeo del psemupro (opcional)
* un plugin de sonido del psemupro (opcional)
* un plugin de cdrom del psemupro (opcional)
* un plugin de intut del psemupro (opcional)

3.2. Control del programa

ePSXe soporta direct input y incluye una opción en el interfaz
gráfico para configurar los pads. Esta es muy simple e intuitiva,
para utilizarla seleccione config->game pads->pad1/pad2
(dependiendo del pad que quiera configurar), y verá una imagen
de un pad, y algunas cajitas con los nombres de las teclas.
Para configurar seleccione el botón de la psx que quiera y
a continuación pulse una tecla del pc para que queden ligados.
(Si lo que quiere es utilizar un joystick pulse un botón del
joystick en vez de una tecla del pc).

Desde la version 2.0 se puede usar un plugin externo para el input
como Pokopom.

Desde la version 1.2.0 puede selecionar que eje analogico usara,
para ello seleccione el eje a configurar y luego use un eje de
su joystick para que este sea capturado y salvado. Ademas puede
configurar el efecto que quiere que realice cada motor mientras
durante el force feedback. 

  ePSXe incluye 8 teclas que cambian rápidamente algunos parámetros:

  - F1-Salva un savestate en el slot seleccionado.
  - F2-Incrementa el slot selecionado. (max = 5)
  - F3-Carga un savestate desde el slot seleccionado.
  - F4-Activa/Desactiva el framelimit.
  - F5-Cambia de pad digital/tipo de pad selecionado.
  - F6-Selecciona sobre que pad actuara la tecla F5.
  - F7-Intercambia on the fly the GPU por la GPUCORE.
  - F8-Hace una foto al directorio snap.

3.4. Configurar el emulador.

Básicamente necesita configurar la bios, el sistema de vídeo, el
sistema de cdrom, el del sonido y los controles para obtener el programa 
funcionando correctamente.

Para video, sonido, cdrom e input, puede selecionar el plugin desde el
menu config->Plugins. Las opciones config->(video/sound/cdrom/gamepads) son
para acceder directamente a la configuracion del plugin seleccionado.

3.4.1. Configurar la bios.

En el interfaz gráfico seleccione config->bios. Aparece una nueva
ventana con una cajita que muestra la bios seleccionada y un botón
a su derecha. Pulse sobre ese botón y seleccione su bios. Se han
testeado encontrándose compatibles las bios scph1001.bin y scph7502.bin,
pero no la scph1000.bin. Por favor recuerde que solo puede poseer una
de esas bios si cumple los requisitos legales establecidos en su país
para ello.

Desde la version 1.9.25, ePSXe incluye una bios HLE propia La compatibilidad es
alta, pero no tanto la precision y las tarjetas de memoria podrían no funcionar
en algunos juegos. 

3.4.2. Configurar el sistema de vídeo.

En el intefaz gráfico seleccione config->plugins->vídeo. Aparece una nueva 
ventana que tiene una caja de selección que muestra el plugin de vídeo(*)
seleccionado, seleccione su plugin de vídeo favorito y configúrelo
con el botón configure y finalmente pulse ok. En la parte inferior
se puede configurar el plugin software distribuido con el emulator.

ePSXe incluye 2 plugins de GPU propios:
1) GPUCORE. Se trata de un plugin software portado desde la version de Android.
Esta plugin requiere OpenGL2 + GLSL shaders para funcionar. Este plugin es muy
preciso y soporta muchas funcionalidades como incrementar la resolucion de
renderizado, shaders, filtros 2D, profiles de brillo ...

2) Pete OpenGL2 Core Plugin. Se trata del plugin Pete GPU OGL2 Plugin integrado
en ePSXe. El plugin requiere una buena tarjeta gráfica (Nvidia/ATI), pero incluye
mucha funcionalidades que permiten mejorar los gráficos del juego.

(*) EPSXE uses PSEmu Pro's video plugins, so you can get other compatible GPU
plugins from psxdatacenter.com

(*) ePSXe usa plugins de vídeo compatibles con el psemupro, asi que puede
conseguir otros plugins en sitios como psxdatacenter.com.

3.4.3. Configurar el sistema de cdrom.

En el intefaz gráfico seleccione config->plugins->cdrom. Aparece una nueva 
ventana que tiene una caja de selección que muestra el plugin de cdrom(*)
seleccionado, seleccione su plugin de cdrom favorito y configúrelo
con el botón configure. Ademas hay una segunda caja de seleccion donde
puede seleccionar su letra de cdrom, si va a usar su primer cdrom
deje seleccionado firstcdrom, sino eliga la letra, finalmente pulse ok.
Se ha probado el emulador principalmente con los plugins internos
de cdrom y con el plugin de cdrom de Pete.

Si utiliza win2k debería utilizar el plugin nativo para win2k
incluido en el emulador.

3.4.4. Configurar el sistema de sonido.

En el intefaz gráfico seleccione config->plugins->sound. Aparece una nueva 
ventana que tiene una caja de selección que muestra el plugin de sonido(*)
seleccionado, seleccione su plugin de cdrom favorito y configúrelo
con el botón configurar. Además en esa ventana puede ver algunas opciones
donde puede activar y desactivar opciones de sonido del emulador.
La playstation tiene 3 tipos de sonido, ADPCM, XA y CDDA:

* Enable Sound  (Select to enable ADPCM sound) - esta opción es usada
para activar el sonido ADPCM, y es necesaria para el sonido XA.

(*) ePSXe usa plugins de sonido compatibles con el psemupro, así necesita
descargarlos primero desde alguna pagina web. Los plugins recomendados
son el de Pete, Eternal y el Interno. Existen otros muy buenos como
Andy o Iori.

3.4.5. Configuring el sistema de input.

En el intefaz gráfico seleccione config->input->sound. Aparece una nueva 
ventana que tiene una caja de selección que muestra el plugin de input
seleccionado, seleccione su plugin de cdrom favorito y configúrelo
con el botón configurar.

Desde ePSXe 2.0 se soportan plugins de input externos como Pokopom.

3.4.6. Eligiendo ficheros de tarjetas de memoria.

En esta ventana, puede seleccionar los ficheros de las tarjetas de
memoria que quiere utilizar. ePSXe soporta la mayoria de los formatos
de memoria mas importantes, .MC(R), .GME, .MCD y .MEM. Puede desde 
esta ventana desactivar las memory cards si así lo desea.

Epsxe por defecto usa 2 tarjetas de memoria compatidas por todos los juegos
pero si lo desea puede activar la opcion que se incluye para que cada juego
tenga sus propias tarjetas de memoria.

3.6. Tarjetas de memoria.

ePSXe tiene soporte para tarjetas de memoria. Estas son
salvadas a dos ficheros en el directorio .\memcards con
los nombre epsxe000.mcr y epsxe001.mcr . Su tamaño
es de 128 Kb cada fichero y el formato usando es plano,
es decir un simple volcado de memoria.

ePSXe no incluye un editor de tarjetas de memoria, pero
puede usar la opción Load->Run BIOS para editar sus
tarjetas de memoria. Si esta usando alguno de los
plugins pete opengl/direct no olvide activar la
el offscreen rendering para ver esto correctamente.

ePSXe 1.0.1 incluye un selector de ficheros de tarjetas de
memoria. ePSXe soporta multiples formatos de tarjetas de
memoria distintos.

3.7. Savestates.

ePSXe 1.2.0 incluye soporte para savestates. Estos son
ficheros que guardan el estado del emulador cuando se
pulsa una tecla, asi podemos restaurar el juego en ese
mismo punto mas tarde, sin necesidad de esperar a que
el juego nos permita salvar.

F1 = Salva el state en el slot seleccionado
F2 = Incrementa el slot selecionado. (Si utilizas el
menu que tienen los plugins durante la ejecucion veras
cual es el slot seleccionado)
(MAX = 5 slots)
F3 = Carga el state del slot seleccionado.

ePSXe 1.2.0, 1.4.0   supports savestates version 0.
ePSXe 1.5.0          supports savestates version 0 and 1.
ePSXe 1.6.0          supports savestates version 0, 1 and 2.
ePSXe 1.7.0          supports savestates version 0, 1, 2 and 3.
ePSXe 1.8.0, 1.9.25  supports savestates version 0, 1, 2 and 3.
ePSXe 2.0.0          supports savestates version 0, 1, 2, 3 and 4.

ePSXe para Windows/Linux soporta los savestates hechos en
Android y vice versa.

3.8. Plugins de juego en red.

ePSXe 1.4.0 incluye soporte para plugins de juego en red.
Para seleccionar el plugin elija config->netplay. Siga
la documentacion incluida con el plugin para su uso.

3.9. Cheat Codes.

ePSXe desde la version 1.8.0 soporte cheat codes formato GameShark. Puede cargar
los cheat codes en el emulador creando un fichero llamado "cheats/SXXX_XXX.XX.txt"
(donde SXXXX_XXX.XX es el nombre PS-EXE del juego, el cual puede ver activando
la consola de logs de ePSXe).
Por ejemplo in Final Fantasy 8 US-NTSC el fichero sería:

"cheats/SLUS_008.92.txt".

ePSXe soporta hasta 256 codigos por fichero. El formato del fichero es:

#descricion del cheat code 1
XXXXXXXX YYYY
ZZZZZZZZ AAAA
#descricion del cheat code 2
NNNNNNNN MMMM

Por ejemplo para Final Fantasy 8 US-NTSC, esto podría ser "cheats/SLUS_008.92.txt":

#Zell level 20
800778a4 4a38

Actualmente, solo los codigos Gameshark de los tipos 80XXXXXX, 30XXXXXX, 50XXXXXX, 
1F800XXX, 10XXXXXX, 11XXXXXX, 20XXXXXX, 21XXXXXX son soportados, el resto serán 
ignorados.
Puede encontrar codigos Gameshark en muchas páginas web por internet, tales como:
http://psxdatacenter.com
http://www.gamegenie.com/cheats/gameshark/ps1/index.html

Después de crear el fichero y ponerlo en "cheats", ejecute el juego y pulse ESC
para desde el menú poder activar/desactivar los cheat codes (desactivados por
defecto en el arranque).

Ora opcion es conseguir el programa PEC (http://pec.emucheater.com) que incluye 
una gran cantidad de trucos :-)

3.10. Soporte de Local netplay (experimental)

Desde la version 2.0.5, ePSXe soporte local netplay sin usar plugins externos.
Esta funcionalidad es compatible con el resto de ports de ePSXe (MacOSX,
Linux, Android ...). Para poder jugar es necesario tener la misma bios y
el mismo juego en ambos dispositivos. El juego debe estar disponible en la
lista de juegos de ambos dispositivos para que pueda ser encontrado.

Como usarlo:

Jugador 1:

1) File>Local netplay>Jugador1
2) Selecciona el juego desde la lista y pulsa el boton ejecutar
3) Ahora puede ver la IP local en la consola, esto será necesario para que
   el jugador pueda conectarse.
4) Espera a que comience el juego.

Jugador 2:

1) File>Local netplay>Jugador2
2) Introduzca la direccion IP local del jugador1
3) Espera a que comiece el juego

Cuando se usa esta funcionalidad las tarjetas de memoria que se usan no son
las normales, sino que hay 2 por juego y se guardan en epsxe/memcards/net.

Además es posible usar savestate desde el jugador 1, y la información será
enviada al jugador 2 por red.

***********************************************************************
4. Recursos implementados
***********************************************************************

- El juego completo de instrucciones del r3000: La PSX
tiene por microprocesador central un r3000.

- Incompleta emulación del coprocessador COP0: cuyo cometido es
controlar ciertas acciones de la consola.

- Emulación del coprocesador GTE: la playstation tiene un coprocesador
matemático que se encarga de calcular la geometría 3d y las luces,
su emulación esta incluida en el emulador pero tiene algunos errores.

- Emulación del puerto de juegos y tarjetas de memoria: ePSXe incluye
emulación del puerto encargado de conectar los pads y las tarjetas
de memoria a la PSX.

- Soporte nativo para ISOS: ePSXe soporta nativamente juegos en
formato cdrom o ficheros iso.

- Soporte para plugins de vídeo programados para el psemupro.

- Soporte para plugins de sonido programados para el psemupro.

- Soporte para plugins de cdrom programados para el psemupro.

- Soporte para plugins de input programados para el psemupro.

- Soporte para plugins de juego en red programados para el psemupro.

- Decodificador MDEC: ePSXe incluye emulación del decodificador
de jpegs de la PSX, recodificada en la version 1.0.1.

- Soporte para sonido CDDA: algunos juegos de PSX utilizan pistas
de audio para guardar el sonido, ePSXe soporta este tipo de sonido.

- Soporte para sonido XA: ePSXe tiene soporte para este tipo de sonido.

- Interfaz gráfico.

Una vez que se intenta ejecutar un programa en ePSXe pueden ocurrir
varias cosas:

     (*) El programa funciona correctamente. Solución: "duro con el
         juego, este no se escapa". Mándenos una imagen cuando acabe
         el juego.

     (*) El juego parece funcionar pero hay fallos gráficos.
         Solución: ePSXe usa los plugins gráficos escritos para el
         psemupro, asi que puede seleccionar uno distinto para ver
         si con ello se soluciona el problema.

     (*) El juego parece funcionar pero hay fallos 3d o con las luces.
         Solución: ePSXe tiene una emulación mas o menos completa del
         GTE, pero a veces se producen fallos, los cuales esperamos
         solucionar en próximas versiones. También podría probar con
         otro plugin para ver si el GTE es realmente el problema.
         (el plugin de pete modo software es el mas compatible)

    (*)  El programa continua sin funcionar. Solución: Si tras todo lo
         anterior el programa no funciona, entonces probablemente no
         sea compatible con ePSXe todavía. Espere una nueva versión o
         pruebe otro emulador.

SONIDO. ePSXe incluye soporte para los plugins de sonido del psemupro
en esta versión, incluyendo sonido ADPCM, CDDA y emulación parcial del XA.
Problemas con el sonido:

    (*)  El programa no suena.
         Solución: asegúrese que tiene activado el sonido en los settings,
         utilizando para ello el interfaz gráfico. Si el sonido sigue sin
         sonar podría tratarse de un juego usando un modo de sonido XA no
         soportado, lo sentimos, intentaremos incluirlo en el futuro.

    (*)  Algunos videos MDEC de Square MDEC tienen un sonido horrible.
         Solución: utilice el plugin de sonido interno, que le dara
         un sonido agradable en los videos.

    (*)  El sonido XA es de baja calidad.
         Solución: Necesitas tener 50/60 frames por segundo para conseguir
         que el sonido sea correcto.

ePSXe v1.5.1, publicado el xx.03.2002

- Incluido soporte parcial de los juegos que utilizan libcrypt. 
  Libcrypt es un esquema de proteccion muy usado en juegos version PAL.
  Este usa el subchannel q. Para poder jugar a estos juegos directamente
  desde tu cdrom, necesitas un cdrom que funcione con las rutinas de
  ePSXe. Pero debes saber que no cada cdrom capaz de leer subchannels
  funcionara en ePSXe. Si el tuyo es uno de los no soportados siempre
  puedes leer el subchannel con clonecd a un fichero .SUB, y copiar el
  fichero al directorio epsxe\patches con el mismo nombre que el 
  ejecutable del juego y con extension .SUB. (SLESXXX.XX.SUB/SCESXXX.XX.SUB/
  SLUSXXX.XX.SUB..) Como leer subchannels es muy lento se incluye una
  opcion para poder salvarlos a disco la primera que ejecutes un juego
  y que el resto de la veces que lo ejecutes no necesites leerlos.
  Por supuesto tambien puedes utilizar plugins de cdrom externos aunque
  de momento solo el de Pete lo soporta. El soporte es parcial porque no
  todos los juegos funcionan, al menos hemos encontrado que el Spryo 2,
  el 3, y el formula one 99 no funcionan.

  En resumen:

  1) Si quiere soporte de suchannels solo desde fichero seleccione en la
     ventana de configuración del cdrom "Enable subchannel support".

  2) Si quiere soporte de subchannels desde el cdrom, seleccione en la
     ventana de configuración del cdrom 
     "Enable subchannel support" y
     "Enable subchannel read from cdrom."
     Si su cdrom es capaz de leer subchannels con epsxe, vera un mensaje
     indicandolo.

  3) Si quiere que se guarden en disco los subchannels la primer vez que
     use un juego, seleccione en la ventana de configuración del cdrom
     "Enable subchannel support",
     "Enable subchannel read from cdrom." y
     "Enable subchannel caching to disk."

  4) Si tiene problemas leyendo cdroms despues de seleccionar 
      "Enable subchannel read from cdrom." probablemente su cdrom no
      se compatible con las rutinas para leer subchannels de ePSXe,
      desactive la opcion y haga sus propios ficheros de subchannels
      con CloneCD.

- Soporte para los lectores de cdrom Teac en los plugins internos
  de cdrom. (Gracias a Pete)

- Incluido soporte para los joystick que mapean sus controles como 
  pov hat. (Gracias a Roberto Paredes por sus multiples pruebas)

- Solucionados algunos problemas en los plugins internos de cdrom,

  evitando que en muchos casos funcionaran los parches ppf.

- Arreglados algunos errores del GTE, que resuelve los problemas de los
  juegos 102 Dalmatas, Rugrats en Paris, Destruction Derby 2, y
  Nuclear Strike.

- Cambiado el timing interno del emulador, lo cual ayuda a que el sonido
  de fondo de algunos juegos vaya a la velocidad correcta (Como ff7,
  Theards of Fate, Valkyrie Profile). Todavia puede utilizar el timing
  de la version anterior si lo desea con el parametro -oldtiming.

- Eliminado el codigo que autodetectaba la version de la bios y realizaba
  modificaciones para conseguir que funcionaran juegos hechos para otras
  bios, lo que evitaba que funcionaran los juegos Legend of Dragon
  y Wild Arms 2 version US.

- Eliminadas las opciones -cdinc y -nocdinc, debidos a ciertos cambios en
  el cdrom decoder ya no son necearias. Ello hace que en esta version el
  parametro -noxaread no tenga alguna utilidad.

- Incluido soporte para transparencias en los MDECs, lo cual hacer que

  opcion solo cuando sea completamente necesaria.
